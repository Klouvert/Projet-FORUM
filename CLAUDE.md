# Arbre Intelligence — Claude Code

Projet civic-tech : forum délibératif visuel sous forme d'arbre botanique.
Docs de référence dans `.claude/` — lire avant de commencer.

## Fichiers de référence

| Fichier | Rôle |
|---|---|
| `.claude/PROJECT.md` | Mission, stack, architecture, conventions durables |
| `.claude/CURRENT_STATE.md` | Sprint courant, fichiers touchés, prochaine étape |
| `.claude/SKILLS/feature-dev.md` | Règles de développement pour ce projet |
| `.claude/SKILLS/debug.md` | Procédure de diagnostic |
| `.claude/SKILLS/review.md` | Checklist de revue |
| `.claude/SKILLS/phase2-mindmap.md` | Spec fonctionnelle Phase 2 (mindmap organique) |

## Convention obligatoire

**En fin de session** (tâche terminée ou interrompue), mettre à jour `.claude/CURRENT_STATE.md` :
- phase / sprint courant
- fichiers modifiés
- problèmes ouverts
- prochaine étape concrète

## Règles essentielles

- Lire un fichier avant de le modifier — jamais supposer son contenu
- Proposer un commit après chaque étape fonctionnelle
- Ne pas enchaîner deux tâches sans validation entre les deux
- Ne pas démarrer la Phase suivante sans confirmation explicite
